History of the Internet

  • What is ARPANet​?
    • Precursor to modern Internet​
  • Who created ARPANet​?
    • Department of Defense​
  • Created in the​ 1960s​
  • NSFNet created in mid-1980s for universities and government labs​
  • ARPANet connected to NSFNet in 1980s​ to create inter-network
  • Use of TCP/IP protocol​ ensured massive scalability

Early roots include the U.S. Dept. of Defense’s ARPANET in the 1960s. In the 1980s, the U.S. National Science Foundation funded a further network “backbone” creation known as NSFNet, connecting universities and government research labs, mostly for email and file transfers (at a time when sending letters and documents via physical mail was the norm). When ARPANET and NSFNet were merged in the late 1980s, the name Internet took over (short for “inter-networking”).

Uniform Resource Locator (URL)​

URLs & DNS & HTTP (15 mins) (slide 9) In order to link pages to one another, each webpage must have a specific address, which is called a Uniform Resource Locator (URL). The components of a URL include the web’s protocol (like HTTPS), the host (the domain you wish to talk to), and the path (the page within the domain you are looking for).

But how does DNS really work?

  1. Browser: Where is www.example.com?
  2. Resolver: Who knows about .com
  3. Root Server: Here is your TLD server’s address
  4. Resolver: Who knows about example.com?
  5. TLD Server: Here is the authoritative server for example.com
  6. Resolver: IP address for www.example.com?
  7. Authoritative Server: 1.2.3.4
  8. Resolver: 1.2.3.4

(slide 11) When you request content from a URL, the first thing that happens is a DNS (Domain Name System) lookup, which takes the domain name from the URL as an input, and replies back with the IP address of the computer that is running the applicable web server. DNS is an example of another protocol!The DNS first looks at the root of the URL (.com, .gov, etc.), then it searches for the domain, and so on.

Hyper Text Transfer Protocol

(slide 14-15) A user uses the web-browser to connect to websites that are “served” by webservers. The hyper-text transfer protocol (“http” in the URLs) in conjunction with other protocols like DNS and TCP/IP help the browser communicate with a webserver. When a “client” requests the contents from a URL, the “server” sends back the HTML which is then displayed by the web browser, a program on our computer which requests the content and renders the HTML into a viewable format for the user.

HTTP is exceptional insecure; it is highly recommended you always use HTTPS whenever you surf the web. In our programming lab in a few lessons you will see first-hand why.

Cloud Computing

  • What is it?
    • Someone else’s computer!
  • Why is it useful?
    • Little to no software installation
    • Applications managed by the provider
    • Lower cost
    • Eliminates the need for physical media
    • Seamless transition between many devices

Cloud Computing (10 mins) Today, we often utilize the processing resources, data, and storage available on the Internet available via different applications. This is called cloud computing and is in contract to local processing and storage that would utilize your specific computers hardware resources. Cloud applications process and deliver data located on remote servers that are accessible via the Internet. Cloud applications have advantages over local drive applications, including:

Little to no software installation: Many applications in the cloud run on remote servers and process or stream data in real-time, thus alleviating users from having to download large software programs that require significant amounts of disk space.

Up-to-date software: Applications running on the cloud are the most recent releases.

Lower cost: Some software applications are very expensive. For example, the Adobe Creative Suite 6 Master Collection, a drawing, photo, and video editing suite, once cost $2,599. Adobe Creative Cloud has moved to a subscription model that offers the same suite of applications for $50 a month, which is cheaper if only needed for a short time.

Eliminates the need for physical media: To listen to music or watch videos on the computer, users could only play physical disks (CDs, DVDs) or media files on the local drive. Current applications like Netflix and Spotify allow users to stream videos and music in the respective online libraries.

Access from many locations and devices: Cloud applications can be accessed by various user devices and from various locations.

ZyBook also discusses the concept of cloud storage, which is the storage of files on remotely-located servers, with those files being managed by apps via the web. Cloud storage has several advantages over storing files solely on a local computer drive:

  • Local storage is limited. Moving lesser-used files to cloud storage, such as large video files, can make room in local storage for more regularly-used files.
  • Files on cloud storage can serve as a backup. A backup is a copy of a file kept in case the local version is accidentally damaged or lost.
  • Files on cloud storage can be accessed by various user devices and from various locations.

Business Models

  • Infrastructure as a Service (IaaS)
    • Pay for access to raw computing resources
    • Amazon Web Services (AWS)
  • Platform as a Service (PaaS)
    • Pay for access to platforms or toolkits
    • Google App Engine
  • Software as a Service (SaaS)
    • Pay for applications
    • Microsoft Office 365

Cloud Computing has also led to the introduction of new business models and impacted the way people conduct business. ZyBook mentions the following:

  • Software as a service (Saas) is a type of business model where users pay for a vendor’s apps, which process and store data on the cloud.

  • Infrastructure as a service (IaaS) is a type of business model where users pay to access computing resources such as servers, storage, and networking. Example: Amazon Web Services (AWS)

  • Platform as a service (PaaS) is a type of business model where users pay to access vendor’s platforms or toolkits to develop and deploy new applications. Example: Google App Engine.

  • Telecommuting, where an employee works outside of the traditional office space, has also increased due to the ability for cloud computing.

Search Engines & Layers of the Web (5 mins)

By now, students should understand the process and protocols in place to request web content and receive it back. But what if the user does not know the URL for their desired content? Where do you go if you don’t know the exact page you want to request? Website such as www.google.com or yahoo, are what is called search engines. A search engine is a website that does web searches – takes user-typed words as an input and finds the web pages that best match.

(Slide 43) A Code.org video explains search engines and search algorithms. (play ~ 0:44 – 3:50)

Key takeaways: - Search engines do not go out to the WWW every time there is a new search to find the best answers - Spiders continuously creates and updates a search index which is then referenced to find the best answers - Algorithms for what is “the best” result(s) can vary and are often evolving to improve results and detect spammers.

Deep and Dark Web

  • Surface Web
    • Publicly accessible World Wide Web
    • Search engine spider can see this!
  • Deep Web
    • Private, unlinked, or hidden
    • Hidden behind login pages
    • Unlinked, excluded by robots.txt
    • Search engines cannot see this!
  • Dark Web
    • Anonymizing protocols
      • Unusual routing techniques
      • Lots of Encryption
    • I2P, Freenet, ZeroNet, GNUnet
    • Commonly accessed using The Onion Router (TOR) browser
  • Do not try this yourself!

(slides 23 - 24) The content that the engine spider indexes makes up the surface web. This is only 4% of the total web content out there! Below the “surface”, is the deep web, content that is hidden behind login pages or proprietary information not accessible to the general public. Then there is the dark web, requiring specific networks, browsers, and software to enable anonymous routing protocols and often used for illegal activity, such as black-market trading and counterfeiting.
